# 詰将棋（難易度3段階・自動生成）

ソルバー内蔵で詰将棋を自動生成する、単一HTMLのアプリです。
すべての問題は「ちょうどその手数で詰むか」をソルバーで検証してから出題します。

## 難易度
画面上部のボタンで切り替えられます。
- **やさしい**：3手詰め（即生成）
- **ふつう**：5手詰め（生成に数秒）
- **むずかしい**：7手詰め（生成に十数秒〜30秒ほど）

長手数ほど生成計算が重いため、バックグラウンド（Web Worker）で先読み生成します。
切り替え直後の1問目は生成待ちになることがあります（「生成中…」表示）。

## ファイル
- `index.html` — アプリ本体（これ1つで完結。外部依存なし）
- `serve_static.js` — ローカル確認用の簡易サーバー（Node製。デプロイには不要）

## ローカルで開く
- 一番簡単：`index.html` をダブルクリック
- うまく動かない場合（ブラウザのWorker制限など）：
  ```
  node serve_static.js
  ```
  を実行し、ブラウザで http://localhost:8123 を開く

## デプロイ
完全な静的サイトなので、`index.html`（と任意でこのフォルダ一式）をそのまま配信するだけです。

- **GitHub Pages**: リポジトリにこのフォルダを置き、Pages のソースに指定するだけ
- **Netlify / Cloudflare Pages**: このフォルダを「公開ディレクトリ」として指定（ビルドコマンド不要）
- **Vercel**: フレームワークなし（Other / static）として、このフォルダを出力にする
- **任意の静的ホスティング / 社内サーバー**: `index.html` をアップロードするだけ

ビルド工程・サーバーサイド処理はありません。

## 仕様メモ
- 攻め方＝先手（盤の手前向き）、難易度に応じて3／5／7手詰め
- 受け方（後手）の持駒は局面ごとに明示し、画面に表示（長手数の生成を現実的にするため「残り全部」ルールは不採用）
- 打ち歩詰め・二歩・行き所のない駒などの反則を考慮
- 受け方は王手を最善で受ける前提で詰みを判定
- 問題はWeb Workerでバックグラウンド先読み生成（Worker不可環境では自動でメインスレッド生成に切替）
